## 문체 프리셋 (10종)

```
프리셋                 │ 파일                        │ 특징
───────────────────────┼────────────────────────────┼──────────────────
🏛️ 정석파 (classic)    │ knowledge/presets/classic.md    │ 안정적, 정부 문서 표준 문체
🔬 기술파 (engineer)   │ knowledge/presets/engineer.md   │ 기술적 디테일 강조
📊 데이터파 (analyst)  │ knowledge/presets/analyst.md    │ 수치/데이터 중심
🚀 비전파 (visionary)  │ knowledge/presets/visionary.md  │ 미래 비전, 성장 스토리
🤝 실적파 (proven)     │ knowledge/presets/proven.md     │ 실적/경험 어필
💡 혁신파 (disruptor)  │ knowledge/presets/disruptor.md  │ 혁신성, 차별화 강조
🌱 사회가치파 (impact) │ knowledge/presets/impact.md     │ 사회적 가치, ESG
🏭 제조파 (builder)    │ knowledge/presets/builder.md    │ 제조/생산 특화
📱 플랫폼파 (scaler)   │ knowledge/presets/scaler.md     │ 플랫폼/확장성 강조
🌏 글로벌파 (global)   │ knowledge/presets/global.md     │ 수출/글로벌 진출
```

**적용 모드:**
- **단일 모드**: 전체 문서에 하나의 프리셋
- **혼합 모드**: 메인 프리셋 + 서브 프리셋 병합
- **섹션별 모드**: 섹션마다 다른 프리셋 (예: 기술 섹션은 engineer, 사업 섹션은 analyst)

## [VISUAL] 태그 규칙

초안 작성 시 시각 자료가 필요한 위치에 다음 형식으로 태그를 삽입:

```markdown
[VISUAL: {유형} — {설명}]
```

**유형 분류:**
```
유형         │ 예시
─────────────┼──────────────────────────────
표 (table)   │ [VISUAL: 표 — 연도별 매출 추이 (2022-2026)]
차트 (chart) │ [VISUAL: 차트 — 시장 규모 성장률 막대그래프]
다이어그램   │ [VISUAL: 다이어그램 — 서비스 아키텍처 플로우차트]
인포그래픽   │ [VISUAL: 인포그래픽 — 핵심 성과 지표 3개 하이라이트]
간트차트     │ [VISUAL: 간트차트 — 12개월 추진일정표]
조직도       │ [VISUAL: 조직도 — 프로젝트 투입 인력 구성]
```

**태그 위치:** 해당 시각 자료가 들어갈 정확한 위치에 단독 줄로 삽입

## 출력물 형식과 저장 위치

| 출력물 | 형식 | 저장 위치 |
|--------|------|-----------|
| 작성 전략서 | Markdown (심사기준별 어필 포인트, 분량 배분, 문체 설정) | `projects/{사업}/context.md` — strategy 섹션 |
| 섹션별 초안 | Markdown (본문 텍스트 + [VISUAL] 태그) | `projects/{사업}/output/drafts/current/{번호}-{섹션명}.md` |
| 수정된 초안 | Markdown | `projects/{사업}/output/drafts/current/{번호}-{섹션명}.md` (업데이트) |
| 백업 초안 | Markdown | `projects/{사업}/output/drafts/v{N}/` (수정 전 백업) |
| 변경 이력 | Markdown (섹션, 변경 내용, 트리거) | `projects/{사업}/context.md` — changelog 섹션 |

### 초안 파일 네이밍 규칙

```
output/drafts/
├── v1/                          # 첫 번째 초안 백업
│   ├── 01-사업개요.md
│   ├── 02-기업현황.md
│   └── ...
├── v2/                          # 수정본 백업
├── current/                     # 현재 최신 버전 (항상 여기서 작업)
│   ├── 01-사업개요.md
│   ├── 02-기업현황.md
│   ├── 03-기술개발내용.md
│   └── ...
```

## 주의사항 / 규칙

### 작성 원칙
- **심사기준 최우선** — 배점 높은 항목에 더 많은 분량과 구체성 할당
- **정량적 표현 우선** — "크게 성장" → "연평균 15.3% 성장 (출처: 통계청, 2025)"
- **구체적 근거 필수** — 모든 주장에는 근거(프로필 데이터, 서류 내용, 공인 출처) 제시
- **추측 금지** — 서류에 있는 내용 기반으로만 작성. 없으면 `[확인 필요]` 태그

### 수치 관련
- **수치 날조 절대 금지** — 출처 없는 수치에는 반드시 `[확인 필요]` 태그
- 표에 사용할 수치는 `[VISUAL]` 태그에 데이터 명시
- 본문 수치와 표/차트 수치가 반드시 일치하도록 작성
- 단위 통일 (천원/백만원/억원), 소수점 자릿수 통일

### 문체 규칙
- 정부 사업계획서 문체 기본 적용
- "~할 것이다" → "~할 계획이며, 이를 통해 ~효과를 기대한다"
- 객관적, 구체적, 수치 기반 서술
- humanizer 스킬로 AI 흔적 제거 (최종 단계)

### 버전 관리
- 수정 시 반드시 current/ → v{N}/ 백업 후 current/ 수정
- 변경 이력은 context.md에 기록 (섹션, 변경 내용, 트리거)
- 연관 섹션 일관성 체크 필수 (예: 예산 변경 → 사업비 총괄표도 수정)

### 호출 조건
- PM이 다음 상황에서 호출:
  - 텍스트 초안 작성 시
  - 섹션 수정 요청 시
  - 작성 전략 수립 시
