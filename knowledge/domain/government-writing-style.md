# 정부 사업계획서 문체 가이드

## 기본 원칙

정부 지원사업 사업계획서는 **심사위원이 읽는 공식 문서**다.
학술 논문도 아니고, 마케팅 자료도 아니다. 그 중간의 '관(官) 문체'가 필요하다.
심사위원은 하루에 수십 건의 사업계획서를 검토하므로, 명확하고 간결한 문체가 핵심이다.
읽히기 쉬우면서도 전문성을 갖추고, 객관적 근거로 뒷받침되는 서술이 요구된다.

---

## 문체 규칙

### 1. 문장 종결

- **~한다/~이다/~있다** 체 사용 (합니다체 금지)
- 예: "본 사업을 통해 생산성을 30% 향상시킨다" (O)
- 예: "본 사업을 통해 생산성을 30% 향상시킵니다" (X)
- 문장 종결이 일관되지 않으면 전체 문서의 품질이 떨어져 보인다
- 같은 문단 내에서 "~이다"와 "~입니다"가 섞이면 감점 사유가 된다
- **예외**: 발표자료(PPT) 또는 요약문에서는 "~합니다"체를 허용하는 사업도 있으나, 본문 계획서는 반드시 "~이다"체로 통일한다

### 2. 주어

- **"본 사업"**, **"당사"**, **"본 과제"** 등 공식 주어 사용
- "저희 회사", "우리" 등 구어체 금지
- 주어 생략 시 문맥상 명확해야 함
- 법인일 경우 "당사", 개인사업자 또는 예비창업자일 경우 "신청인" 또는 "대표자"를 사용한다
- 연구개발 과제에서는 "주관기관", "참여기관", "위탁기관" 등 공고문에서 정의한 용어를 그대로 사용한다
- 같은 주체를 가리킬 때 문서 전체에서 동일한 호칭을 사용한다 (예: "당사"로 시작했으면 끝까지 "당사")

### 3. 수치 표현

- 구체적 수치 + 근거 필수: "국내 시장 규모는 약 2조 5,000억 원이다(출처: 통계청, 2025)"
- 모호한 표현 금지: "많은", "상당한", "크게 증가" → 구체적 수치로 대체
- 단위 통일: 한 문서 내에서 천원/백만원/억원 중 하나로 통일
- 소수점: 백분율은 소수점 1자리, 금액은 정수
- 연도 표기: "2025년" 또는 "'25년"으로 통일하되, 한 문서 내 혼용 금지
- 천 단위 구분은 쉼표(,) 사용: "1,250백만 원"
- 성장률/감소율 표기 시 기준 연도를 반드시 명시: "전년 대비 15.3% 성장(2024→2025)"
- **출처 기재 원칙**: 3년 이내 자료를 사용하며, 가능한 1차 출처(통계청, 산업연구원 등 공인기관)를 인용한다

### 4. 문단 구성

- 한 문단 = 한 핵심 메시지
- 두괄식 (핵심 → 설명 → 근거)
- 문단 길이: 3~5문장 적정
- 불릿 포인트와 서술형 적절히 혼합
- 서술형 문단 후 표/그림으로 요약하면 가독성이 높아진다
- 각 섹션 시작 시 한 문장으로 해당 섹션의 핵심 내용을 요약하는 것이 효과적이다
- 문단 간 전환어를 적절히 사용한다: "이에 따라", "한편", "특히", "또한" 등

### 5. 피해야 할 표현

| 피할 것 | 대체 표현 | 이유 |
|---------|----------|------|
| 최고의, 최첨단의 | 높은 수준의, 선도적인 | 검증 불가능한 최상급 표현 |
| 혁신적인 (남발) | 차별화된, 개선된 | 과용 시 신뢰도 하락 |
| ~할 것입니다 | ~할 계획이다, ~을 추진한다 | 합니다체 금지 |
| 매우, 아주, 굉장히 | (삭제하거나 수치로 대체) | 불필요한 수식어 |
| 기존에 없던 | 기존 대비 차별화된 | 사실 검증이 어려운 표현 |
| ~에 있어서 | ~에서, ~에 대해 | AI 특유의 관용 표현 |
| ~라 할 수 있다 | ~이다 (직접 서술) | 약한 표현, 확신 부족 |
| 패러다임 전환 | 구조적 변화, 전환 | 과장된 표현 |
| 시너지 효과 | 상호 보완 효과, 연계 효과 | 남용되는 표현 |
| 차세대 | 후속, 개량형, 고도화된 | 구체성 부족 |
| ~것은 자명하다 | ~것으로 확인된다 | 학술적 과잉 표현 |

### 6. 강조 기법

- **볼드체**: 핵심 키워드, 핵심 수치에만 사용 (한 문단에 2~3개 이내)
- 밑줄: 사용 자제 (가독성 저하)
- 색상: 표의 헤더나 강조 셀에만 제한적 사용
- 글자 크기 변화: 양식에서 지정한 크기 외 임의 변경 금지
- 음영(배경색): 핵심 요약 박스에 제한적으로 사용 가능
- 볼드체를 과다 사용하면 오히려 아무것도 강조되지 않는 효과가 생긴다

### 7. 전문 용어 사용

- 처음 등장 시 한글(영문 약어) 형태로 표기: "사물인터넷(IoT)"
- 이후부터는 약어만 사용 가능
- 업계에서 통용되지 않는 자체 약어 사용 금지
- 공고문에서 사용한 용어를 그대로 사용하는 것이 가장 안전하다
- 심사위원이 해당 분야 비전문가일 수 있으므로, 핵심 기술 용어는 간략한 설명을 첨부한다

### 8. 표 작성 원칙

- 표 제목은 상단에, "[표 N] 제목" 형식으로 기재한다
- 헤더 행은 배경색(회색 등)으로 구분한다
- 셀 내 텍스트는 좌측 정렬, 숫자는 우측 정렬
- 단위는 표 제목이나 헤더에 명시한다 (예: "(단위: 백만 원)")
- 합계 행은 굵은 글씨 또는 배경색으로 구분한다
- 빈 셀은 "-" 또는 "해당 없음"으로 채운다

## 섹션별 문체 특징

### 사업 개요
- 간결하고 명확하게, 사업의 필요성과 목적을 1~2문단으로
- "~의 필요성이 대두되고 있다" → "~가 시급하다/필요하다"
- 개요 첫 문장에서 "무엇을 왜 하는가"가 한 번에 드러나야 한다
- 정부 정책 키워드를 자연스럽게 녹여야 한다 (예: "디지털 전환", "탄소중립", "수출 다변화")

### 기술 개발 내용
- 기술적 구체성 + 도식/표 병행
- 전문 용어는 괄호 안 설명 추가
- "본 기술은 A 방식을 적용하여 B 문제를 해결한다"
- 기술 흐름도(플로우차트)나 시스템 구성도를 반드시 포함한다
- 개발 단계별 목표와 산출물을 표로 정리하면 효과적이다
- 기존 기술 대비 개선점을 정량적으로 비교하는 표를 포함한다

### 사업화 전략
- 시장 데이터 기반 논거
- TAM → SAM → SOM 순서로 시장 규모 제시
- 매출 추정은 보수적 시나리오 포함
- BM(비즈니스 모델)을 명확하게 서술한다: 누구에게 무엇을 어떻게 판매하는가
- 경쟁사 분석은 2~3개 이상 구체적 비교 (기능 비교표 형태 권장)
- 수익 모델은 단가 x 수량 기반의 산출 근거를 명시한다
- 시장 진입 전략(초기 고객 확보 방안)을 구체적으로 기술한다

### 대표자 역량
- 팩트 기반 (경력, 특허, 수상, 인증)
- 자기 자랑 아닌 객관적 실적 나열
- 학력보다 관련 업무 경험과 실적을 중심으로 기술한다
- 팀 역량의 경우 핵심 인력 3~5인의 경력을 표로 정리한다
- 해당 기술/산업 분야와의 관련성을 명확히 드러낸다

### 추진 일정
- 간트차트(Gantt Chart) 형태로 시각화한다
- 월 단위 또는 분기 단위로 마일스톤을 구분한다
- 각 단계별 산출물(deliverable)을 명시한다
- 시작-종료 시점이 다른 섹션(예: 예산)과 일치해야 한다

## 분량 가이드

- 글자 수 제한이 있으면 90~95% 채우기 (100% 넘기면 잘림)
- 페이지 제한이 있으면 마지막 페이지 70% 이상 채우기
- 빈 칸 없이 모든 항목 작성 (해당 없음 시 "해당 없음" 명시)
- 글자 수 제한이 없는 항목이라도 지나치게 짧으면 성의 없어 보인다
- 표와 그림은 분량 채우기용이 아닌, 내용 전달의 효율성을 위해 사용한다

## 문장 다듬기 체크리스트

1. 한 문장이 3줄(약 80자)을 넘기지 않는가?
2. 주어와 술어가 호응하는가?
3. 불필요한 수식어("매우", "상당히" 등)가 없는가?
4. 같은 단어가 한 문단에서 3회 이상 반복되지 않는가?
5. 능동태로 쓸 수 있는 문장을 수동태로 쓰지 않았는가?
6. 모든 수치에 출처 또는 산출 근거가 있는가?
7. 외래어/영문 표기가 국립국어원 표기법을 따르는가?
8. 접속사가 남발되지 않았는가? (한 문장에 "그리고", "또한" 중복 사용 금지)
