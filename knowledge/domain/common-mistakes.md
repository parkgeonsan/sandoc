# 사업계획서 흔한 실수 목록

## 개요

사업계획서 작성 시 자주 발생하는 실수들을 유형별로 정리한다.
이 목록은 자가 심사(review) 단계에서 체크리스트로 활용한다.

---

## 1. 양식/서식 관련

- [ ] **양식 서식 변경** — 원본 양식의 글꼴, 크기, 줄간격을 임의로 변경
- [ ] **빈칸 방치** — 작성할 내용이 없는 항목을 빈칸으로 남김 ("해당 없음" 기재 필요)
- [ ] **페이지 초과** — 지정된 페이지 수를 초과하여 작성 (잘리거나 감점)
- [ ] **표 서식 깨짐** — 셀병합 오류, 표 경계 누락, 내용 잘림
- [ ] **이미지 해상도 부족** — 차트/다이어그램이 흐릿하게 출력됨

---

## 2. 내용/논리 관련

- [ ] **추상적 서술** — "혁신적인 기술로 시장을 선도할 것이다" (구체성 없음)
- [ ] **논리 비약** — 문제 제기 → 갑자기 결론 (해결 방법 누락)
- [ ] **반복 서술** — 같은 내용을 여러 섹션에서 반복 (지면 낭비)
- [ ] **섹션 간 불일치** — 사업 개요와 기술 내용이 다른 이야기
- [ ] **정책 키워드 누락** — 공고문에 강조된 정책 방향을 사업계획서에 반영하지 않음

---

## 3. 수치/데이터 관련

- [ ] **출처 없는 수치** — 시장 규모, 성장률 등 근거 없이 제시
- [ ] **수치 불일치** — 본문의 수치와 표/차트의 수치가 다름
- [ ] **합계 오류** — 세부 항목 합계 ≠ 총계
- [ ] **단위 불통일** — 같은 문서에서 천원, 백만원, 억원 혼용
- [ ] **비현실적 매출 추정** — 근거 없이 매년 300% 성장 예측
- [ ] **예산 부적정** — 인건비 비율 과다, 재료비 산출 근거 없음

---

## 4. 문체/표현 관련

- [ ] **합니다체/이다체 혼용** — 문장 종결 스타일이 섞임
- [ ] **과장 표현 남발** — "최초", "최고", "유일무이" 등 검증 불가 표현
- [ ] **AI 특유 표현** — "~에 있어서", "~라 할 수 있다", "~것은 자명하다" 반복
- [ ] **구어체 혼입** — "되게", "엄청", "좀" 등
- [ ] **오탈자** — 회사명, 사업명, 전문 용어 오타

---

## 5. 전략적 실수

- [ ] **고배점 항목 소홀** — 30점짜리 기술성 항목에 반 페이지만 작성
- [ ] **가점 누락** — 특허, 인증, 고용 계획 등 가점 요소를 챙기지 않음
- [ ] **심사기준 무시** — 공고문 심사기준과 무관한 내용 나열
- [ ] **차별화 부재** — 경쟁사 대비 왜 우리인지에 대한 답 없음
- [ ] **사업비와 내용 불일치** — 개발 내용에 없는 항목이 예산에 있음

---

## 6. 제출 관련

- [ ] **필수 서류 누락** — 요구된 첨부 서류 빠뜨림
- [ ] **파일 형식 오류** — HWP로 내야 하는데 PDF로 제출
- [ ] **파일명 미준수** — 공고에서 지정한 파일명 규칙 미적용
- [ ] **제출 기한 초과** — 마감 시간 경과 후 제출 시도

---

## 자가 심사 시 활용법

1. 초안 완성 후 위 체크리스트를 순회하며 점검
2. 해당되는 항목이 있으면 수정 후 재확인
3. 특히 **수치 불일치**와 **빈칸 방치**는 자동 검증 가능
4. 심사기준 매핑과 대조하여 고배점 항목 충실도 확인
