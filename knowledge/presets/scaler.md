# 플랫폼파 (Scaler) 프리셋

## 프리셋 이름
**📱 플랫폼파 (Scaler)**

## 설명
**플랫폼 비즈니스**와 **SaaS(Software as a Service) 스케일링 전략**을 중심으로 사업계획서를 구성하는 문체이다. 네트워크 효과, 사용자 성장 지표, 구독 경제 모델, 데이터 기반 의사결정 등 플랫폼·SaaS 고유의 언어와 KPI를 전면에 배치한다. "단일 제품 판매"가 아닌 "확장 가능한 시스템 구축"의 관점에서 사업을 서술하며, 사용자 획득-활성화-유지-수익화-추천(AARRR)의 그로스 프레임워크를 축으로 성장 전략을 논증한다. 한계 비용 체감, 네트워크 효과에 의한 비선형 성장, 데이터 자산 축적이 핵심 경쟁력임을 강조한다.

---

## 핵심 특징

1. **네트워크 효과 논증**: 사용자 수 증가가 서비스 가치를 높이는 선순환 구조를 명확히 서술한다. 직접 네트워크 효과(같은 유형 사용자 간)와 간접 네트워크 효과(공급자-수요자 간)를 구분하여 제시한다.
2. **AARRR 기반 성장 지표**: Acquisition(획득), Activation(활성화), Retention(유지), Revenue(수익), Referral(추천)의 퍼널별 KPI를 정량적으로 제시하며, 각 단계의 목표치와 개선 전략을 기술한다.
3. **구독·반복 매출 모델**: MRR(월 반복 매출), ARR(연 반복 매출), Churn Rate(이탈률), LTV/CAC 비율 등 SaaS 고유 재무 지표를 활용하여 사업의 수익성과 지속가능성을 논증한다.
4. **기술 아키텍처의 확장성**: 마이크로서비스, 클라우드 네이티브, API-first, 멀티테넌시 등 기술적 확장성을 뒷받침하는 아키텍처 설계를 구체적으로 서술한다.
5. **데이터 자산 전략**: 서비스 운영을 통해 축적되는 데이터가 어떻게 서비스 개선, 신규 수익원 창출, 진입장벽 구축에 활용되는지를 체계적으로 기술한다.

## 권장 표현 예시

### 사업 개요
- "본 사업은 [산업] 분야의 [공급자]와 [수요자]를 연결하는 양면(Two-sided) 플랫폼을 구축하는 과제이다. 플랫폼의 네트워크 효과를 통해, 사용자 수 증가 → 데이터 축적 → 서비스 품질 향상 → 추가 사용자 유입의 선순환 구조를 형성한다."
- "당사의 [서비스명]은 SaaS 모델로 제공되며, 월 구독료 기반의 안정적 반복 매출(MRR) 구조를 갖추고 있다. 현재 MRR [X]만 원에서 본 사업을 통해 [연도]까지 MRR [Y]만 원 달성을 목표로 한다."
- "본 플랫폼의 핵심 가치는 [문제]를 [기존 방식] 대비 [X]% 효율적으로 해결하는 데 있으며, 이미 베타 서비스에서 [Y]명의 사용자가 검증한 바 있다(MAU [Z]명, DAU [W]명)."
- "본 사업은 기존 [산업]의 가치사슬을 디지털화하여 중간 단계를 [X]개에서 [Y]개로 축소하고, 이 과정에서 발생하는 비효율([Z]%)을 제거하는 것을 목표로 한다."

### 기술 개발
- "시스템 아키텍처는 클라우드 네이티브(AWS/GCP) 기반 마이크로서비스로 설계하며, 서비스 간 독립 배포·확장이 가능하다. 현재 트래픽 대비 [X]배까지 자동 스케일링이 가능한 구조로 설계한다."
- "API-first 전략을 채택하여 외부 시스템과의 연동성을 확보한다. RESTful API [X]개, Webhook [Y]개를 제공하며, 개발자 포털을 통해 API 문서와 SDK를 공개하여 파트너 생태계 구축을 지원한다."
- "멀티테넌시(Multi-tenancy) 아키텍처를 적용하여 고객사별 독립 환경을 제공하면서도, 인프라 운영 비용을 공유하여 한계 비용을 최소화한다. 고객 1개사 추가 시 한계 비용은 약 [X]원으로, 매출 대비 [Y]% 수준이다."
- "데이터 파이프라인은 실시간 수집 → 전처리 → 분석 → 인사이트 도출의 자동화 구조로 구축하며, 축적된 데이터를 기반으로 [추천 알고리즘/예측 모델/자동화 기능]을 지속적으로 고도화한다."

### 사업화 전략
- "그로스 전략은 AARRR 프레임워크에 기반한다: 획득(Acquisition) — [채널]을 통한 월 [X]명 신규 가입 목표, 활성화(Activation) — 가입 후 [Y]일 내 핵심 기능 사용률 [Z]% 목표, 유지(Retention) — 월간 유지율 [W]% 목표, 수익(Revenue) — 유료 전환율 [V]% 목표, 추천(Referral) — 바이럴 계수(K-factor) [U] 이상 목표."
- "가격 전략은 프리미엄(Freemium) 모델을 채택한다: Free 플랜([기능 제한], 사용자 유입용), Basic 플랜(월 [X]만 원, [핵심 기능]), Pro 플랜(월 [Y]만 원, [고급 기능 + 지원]), Enterprise 플랜(별도 협의, [맞춤 기능 + SLA])."
- "LTV/CAC 분석: 고객 생애 가치(LTV)는 평균 구독 기간 [X]개월 × 월 구독료 [Y]만 원 = [Z]만 원이며, 고객 획득 비용(CAC)은 [W]만 원으로, LTV/CAC 비율 [V]배이다. SaaS 업계 권장 기준(3배 이상)을 충족한다."
- "플랫폼 성장 단계: [연도] Phase 1 — 공급자 [X]개사 온보딩 및 수요자 [Y]명 확보(치킨-에그 문제 해결), [연도] Phase 2 — 네트워크 효과 본격화(임계 사용자 수 [Z]명 돌파), [연도] Phase 3 — 데이터 기반 부가서비스 출시 및 수익 다각화."

### 대표자 역량
- "대표자는 [플랫폼/SaaS 기업]에서 [X]년간 [직책]으로 근무하며, 서비스를 MAU [Y]명에서 [Z]명으로 성장시킨 경험을 보유하고 있다."
- "CTO는 [기업]에서 일 [X]만 건 이상의 트래픽을 처리하는 시스템을 설계·운영한 경력이 있으며, 대규모 트래픽 대응 및 시스템 안정성 확보에 전문성을 갖추고 있다."
- "그로스 팀은 PM(경력 [X]년), 데이터 분석가(경력 [Y]년), 그로스 마케터(경력 [Z]년)로 구성되어, 데이터 기반 성장 전략 수립 및 실행 역량을 갖추고 있다."
- "당사 기술팀은 풀스택 개발자 [X]명, 백엔드/인프라 전문가 [Y]명, 데이터 엔지니어 [Z]명, UX 디자이너 [W]명으로 구성되며, 2주 단위 스프린트로 평균 [V]건의 기능 릴리스를 수행하고 있다."

## 적합한 사업 유형

- **TIPS (민간투자주도형 기술창업지원)**: 기술 기반 플랫폼/SaaS 스타트업에 최적
- **ICT 융합 산업 지원사업 (과기정통부)**: 디지털 플랫폼 기반 산업 혁신 과제
- **초기창업패키지/예비창업패키지**: IT 기반 플랫폼 스타트업
- **창업도약패키지**: 초기 PMF를 검증한 플랫폼의 스케일업 지원
- **데이터·AI 활용 지원사업**: 데이터 기반 플랫폼 고도화
- **클라우드 서비스 지원사업**: SaaS 기반 서비스 개발·운영
- **디지털 서비스 혁신사업**: 전통 산업의 디지털 전환 플랫폼
- **규제 샌드박스**: 기존 규제 프레임에 없는 플랫폼 비즈니스

## 실제 사용 가능한 문장 템플릿

### 사업 개요 섹션

1. "본 사업은 [산업] 분야에서 [공급자 유형] [X]만 개사와 [수요자 유형] [Y]만 명을 연결하는 양면 플랫폼을 구축하는 과제이다. [기존 방식]의 비효율(평균 [Z]시간/건, 수수료 [W]%)을 디지털화하여 [처리 시간] [A]분/건, [수수료] [B]%로 개선한다."

2. "당사 [서비스명]의 현재 핵심 지표: 가입자 [X]명(전월 대비 +[Y]%), MAU [Z]명(전월 대비 +[W]%), 유료 전환율 [V]%, MRR [U]만 원, 월간 이탈률(Churn) [T]%. 본 사업을 통해 [연도]까지 MAU [S]명, MRR [R]만 원 달성을 목표로 한다."

3. "플랫폼의 네트워크 효과 메커니즘: [공급자] 수 증가 → [수요자]의 선택지 확대 → [수요자] 유입 증가 → [공급자]의 매출 기회 확대 → 추가 [공급자] 유입. 이 선순환이 본격화되는 임계점은 [공급자] [X]개사, [수요자] [Y]명으로 분석하며, [연도] [분기]에 도달할 것으로 추정한다."

### 기술 개발 섹션

4. "시스템 아키텍처는 [클라우드 프로바이더] 기반 마이크로서비스로 구성한다. 핵심 서비스: (1) [서비스 A] — [기능], (2) [서비스 B] — [기능], (3) [서비스 C] — [기능]. 서비스 간 통신은 [메시지 큐/API Gateway]를 통해 비동기 처리하며, 개별 서비스의 독립 배포·스케일링이 가능하다."

5. "SLA(Service Level Agreement) 목표: 가용성 [X]% 이상(월간 다운타임 [Y]분 이내), 평균 응답 시간 [Z]ms 이내(P95 기준 [W]ms 이내), 데이터 백업 주기 [V]시간, RPO(Recovery Point Objective) [U]시간, RTO(Recovery Time Objective) [T]시간."

6. "데이터 자산 전략: 서비스 운영을 통해 [데이터 유형 A](월 [X]만 건 축적), [데이터 유형 B](월 [Y]만 건 축적), [데이터 유형 C](월 [Z]만 건 축적)의 고유 데이터를 축적한다. 이 데이터를 기반으로 (1) [추천/매칭] 알고리즘 고도화, (2) [산업 인사이트] 리포트 판매, (3) [예측 모델] API 서비스를 단계적으로 출시한다."

### 사업화 전략 섹션

7. "수익 모델은 3개 축으로 구성한다: (1) 구독료(전체 매출의 [X]%) — [요금제별 설명], (2) 거래 수수료(전체 매출의 [Y]%) — 플랫폼 내 거래 건당 [Z]%, (3) 부가서비스(전체 매출의 [W]%) — [프리미엄 기능], [데이터 분석], [광고]. 매출 구성은 사업 초기 구독료 중심에서 성장기 거래 수수료 중심으로 전환될 것으로 전망한다."

8. "SaaS 유닛 이코노믹스: 월 ARPU(사용자당 평균 매출) [X]만 원, 평균 구독 기간 [Y]개월, LTV(고객 생애 가치) [Z]만 원. CAC(고객 획득 비용)은 [채널 A] [W]만 원, [채널 B] [V]만 원이며, 가중 평균 CAC [U]만 원 기준 LTV/CAC 비율은 [T]배이다. CAC 회수 기간(Payback Period)은 [S]개월이다."

9. "그로스 해킹 전략: (1) 획득 — [전략: 콘텐츠 마케팅/SEO/파트너십], 목표 월 [X]명 신규 가입, (2) 활성화 — 온보딩 프로세스 최적화, 가입 후 [Y]일 내 핵심 액션 완료율 [Z]% 목표, (3) 유지 — [기능: 알림/리포트/커뮤니티]를 통한 재방문 유도, D30 유지율 [W]% 목표, (4) 추천 — 인앱 추천 프로그램, 바이럴 계수(K-factor) [V] 이상 목표."

### 대표자 역량 섹션

## 문체 가이드라인

### 톤 & 보이스
- **데이터 드리븐 어조**: 모든 주장을 사용자 지표, 재무 지표, 기술 지표로 뒷받침한다
- **성장 중심 서술**: 현재 상태보다 성장률과 성장 잠재력에 초점을 맞춘다
- **구조화된 프레임워크 활용**: AARRR, LTV/CAC, 네트워크 효과 등 검증된 프레임워크로 논증한다
