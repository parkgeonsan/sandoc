# 사업계획서 섹션별 작성 패턴

## 개요

정부 사업계획서의 주요 섹션별 작성 패턴을 정리한다.
각 섹션의 목적, 구조, 필수 포함 요소, 흔한 실수를 정의하여
일관성 있고 고품질의 사업계획서를 생성하는 데 활용한다.

---

## 1. 기업 개요 (Company Overview)

### 목적
심사위원에게 기업의 기본 정보와 현황을 신속하게 전달한다.

### 권장 구조
```
1. 기업 기본 정보 (표 형식)
   - 기업명, 대표자, 설립일, 업종, 소재지 등
2. 주요 사업 내용
   - 핵심 제품/서비스 1~2문단 설명
3. 기업 연혁 (핵심 이정표만)
   - 설립, 주요 인증, 핵심 매출 달성, 투자 유치 등
4. 조직 현황
   - 인원 구성 (기술/영업/관리)
5. 재무 현황 요약
   - 최근 3년 매출/영업이익 (표)
```

### 필수 포함 요소
- [ ] 기업 기본 정보 표
- [ ] 대표자 인적사항
- [ ] 주요 제품/서비스 설명
- [ ] 조직 구성 (인원수)
- [ ] 재무 현황 (매출, 영업이익)

### 흔한 실수
- 불필요하게 긴 연혁 나열 (핵심만 5~7개)
- 매출 수치 없이 "성장 중" 표현
- 기업 소개와 사업 아이템 설명 혼동

## 2. 문제 인식 (Problem Recognition)

### 목적
본 사업이 해결하고자 하는 문제의 심각성과 시급성을 설득력 있게 전달한다.

### 권장 구조
```
1. 문제 정의
   - 어떤 문제가 존재하는가? (1~2문장 명확 정의)
2. 문제의 규모와 심각성
   - 피해 규모, 영향 범위를 수치로 제시
3. 기존 해결 방식의 한계
   - 현재 어떻게 대응하고 있으며, 왜 부족한가
4. 해결의 시급성
   - 왜 지금 해결해야 하는가 (정책, 시장, 기술 변화)
5. 본 사업의 해결 방향 (Preview)
   - 어떤 접근으로 해결할 것인가 (1~2문장)
```

### 필수 포함 요소
- [ ] 구체적 문제 정의
- [ ] 문제의 정량적 규모 (피해액, 영향 인구 등)
- [ ] 기존 방식의 한계 분석
- [ ] 정책/시장 관점의 해결 시급성
- [ ] 출처가 명시된 데이터

### 흔한 실수
- 문제를 정의하지 않고 바로 솔루션 설명
- "모두가 겪는 문제" 식의 모호한 서술
- 데이터 없이 심각성만 주장

---

## 3. 목표 시장 분석 (Target Market)

### 목적
사업의 시장성과 성장 잠재력을 객관적 데이터로 입증한다.

### 권장 구조
```
1. 글로벌 시장 현황
   - 규모, 성장률(CAGR), 주요 동향
2. 국내 시장 현황
   - 규모, 성장률, 글로벌 대비 비중
3. TAM-SAM-SOM 분석
   - 각 단계별 산출 근거 명시
4. 목표 고객 세그먼트
   - 1차/2차/3차 타겟 고객 정의
5. 시장 진입 전략
   - 초기 시장 공략 방안
6. 경쟁 환경 분석
   - 주요 경쟁사 비교표 (기능, 가격, 성능)
```

### 필수 포함 요소
- [ ] 시장 규모 수치 (출처 명시)
- [ ] CAGR (연평균 성장률)
- [ ] TAM → SAM → SOM 산출
- [ ] 경쟁사 비교표 (최소 3개사)
- [ ] 타겟 고객 정의
- [ ] 시장 진입 전략

### 흔한 실수
- 글로벌 TAM만 크게 제시하고 SOM 산출 근거 없음
- 경쟁사를 나열만 하고 비교 분석 없음
- 출처 불명의 시장 데이터
- "블루오션" 주장 (경쟁 부재를 장점으로 제시)

---

## 4. 솔루션/기술 개발 내용 (Solution / R&D Plan)

### 목적
핵심 기술의 독창성, 실현 가능성, 개발 계획을 체계적으로 전달한다.

### 권장 구조
```
1. 솔루션 개요
   - 무엇을 어떻게 해결하는가 (핵심 1~2문장)
2. 기술 아키텍처
   - 시스템 구성도, 기술 블록 다이어그램
3. 핵심 기술 요소
   - 각 기술 요소별 상세 설명 및 차별점
4. 기존 기술 대비 비교
   - 정량적 성능 비교표
5. 개발 현황 (TRL)
   - 현재 기술 수준과 목표 수준
6. 개발 계획 (단계별)
   - 연차별/단계별 개발 목표와 산출물
7. 지식재산권 현황 및 전략
   - 보유 특허, 출원 계획
```

### 필수 포함 요소
- [ ] 기술 아키텍처/구성도
- [ ] 핵심 기술의 차별점 (정량적)
- [ ] 기존 기술 대비 비교표
- [ ] TRL 현재 → 목표
- [ ] 단계별 개발 계획과 산출물
- [ ] 특허/IP 현황

### 흔한 실수
- 기술 원리 설명 없이 "혁신적" 수식어만 사용
- 비교 대상 없는 성능 수치
- 개발 계획이 지나치게 추상적
- 비전문가를 위한 요약 누락

## 5. 비즈니스 모델 (Business Model)

### 목적
수익 창출 구조와 사업의 경제적 지속가능성을 논증한다.

### 권장 구조
```
1. 수익 모델
   - 누구에게, 무엇을, 어떻게 판매하는가
2. 가격 전략
   - 가격 설정 근거, 경쟁사 대비 포지셔닝
3. 판매 채널
   - 직판/대리점/온라인/B2G 등
4. 매출 추정
   - 보수적/기본/낙관적 3개 시나리오
   - 단가 x 수량 기반 산출 근거
5. 원가 구조
   - 변동비/고정비 분석
6. 손익분기점(BEP)
   - 월/연 BEP 산출
```

### 필수 포함 요소
- [ ] 수익 모델 명확 정의
- [ ] 매출 추정 (3개 시나리오)
- [ ] 단가 x 수량 산출 근거
- [ ] BEP 분석
- [ ] 원가 구조

### 흔한 실수
- "다양한 수익 모델" 나열만 하고 주력 모델 불명확
- 근거 없는 매출 추정 (매년 200~300% 성장)
- 원가/마진 분석 누락

## 6. 사업화 전략 (Go-to-Market Strategy)

### 목적
제품/서비스의 시장 출시와 확산 전략을 구체적으로 제시한다.

### 권장 구조
```
1. 시장 진입 전략
   - 1차 타겟 시장 및 진입 방법
2. 마케팅/영업 전략
   - 고객 획득 채널, 마케팅 방법
3. 단계별 확산 계획
   - 1단계(국내) → 2단계(확대) → 3단계(해외)
4. 파트너십/협력 전략
   - MOU, 제휴, 유통 파트너
5. 해외 진출 계획
   - 대상국, 인증, 현지화 전략
6. 고객 확보 현황
   - PoC, 파일럿, MOU, 납품 실적
```

### 필수 포함 요소
- [ ] 단계별 시장 확산 전략
- [ ] 초기 고객 확보 방안
- [ ] 마케팅/영업 채널
- [ ] 파트너십/협력 현황
- [ ] 해외 진출 계획 (해당 시)

---

## 7. 팀 구성 및 역량 (Team)

### 목적
과제 수행에 필요한 인적 역량을 갖추고 있음을 입증한다.

### 권장 구조
```
1. 대표자 프로필
   - 경력, 전문성, 관련 실적
2. 핵심 인력 현황 (표)
   - 이름, 직위, 전공, 경력, 역할
3. 조직 구성도
   - 부서별 인원 및 역할
4. 인력 확보 계획
   - 추가 채용 계획 (직무, 인원, 시기)
5. 자문단/멘토
   - 외부 전문가 구성
```

### 필수 포함 요소
- [ ] 대표자 이력 (경력, 학력, 수상)
- [ ] 핵심 인력 표 (3~5인)
- [ ] 역할 분담 (기술개발 내용과 매칭)
- [ ] 채용 계획

## 8. 추진 일정 (Timeline)

### 목적
사업의 추진 가능성과 실행력을 일정 계획으로 증명한다.

### 권장 구조
```
1. 전체 추진 일정 (간트차트)
   - 월/분기 단위 마일스톤
2. 단계별 상세 계획
   - 각 단계의 수행 내용, 산출물, 성과 지표
3. 핵심 마일스톤
   - Go/No-Go 기준점
4. 위험 관리 계획
   - 예상 리스크와 대응 방안
```

### 필수 포함 요소
- [ ] 간트차트 (시각화)
- [ ] 단계별 산출물 명시
- [ ] 마일스톤 정의
- [ ] 일정과 예산의 일관성

### 흔한 실수
- 모든 활동이 병렬로 진행되는 비현실적 일정
- 산출물 없는 마일스톤
- 일정과 예산 불일치

---

## 9. 사업비 예산 (Budget)

### 목적
사업비 편성의 합리성과 적정성을 입증한다.

### 권장 구조
```
1. 사업비 총괄표
   - 정부출연금/민간부담금(현금+현물) 구분
2. 비목별 상세 내역
   - 인건비, 재료비, 외주비, 장비비 등
3. 연차별 예산 배분
   - 1차/2차 연도별 비목별 금액
4. 산출 근거
   - 각 항목의 단가 x 수량 근거
```

### 필수 포함 요소
- [ ] 총괄 예산표 (비목별 합계 일치)
- [ ] 인건비 산출 근거 (등급, 참여율, 개월수)
- [ ] 재료비 산출 근거 (품명, 규격, 수량, 단가)
- [ ] 개발 내용과 예산 항목의 1:1 대응
- [ ] 정부출연금/민간부담금 비율 적정성

### 흔한 실수
- 합계 불일치 (세부 ≠ 소계 ≠ 총액)
- 인건비 비율 70% 이상 (과다)
- 개발 내용에 없는 항목이 예산에 존재
- 간접비 누락 또는 비율 오류

## 범용 패턴: 섹션 작성 공통 원칙

### 두괄식 구조
모든 섹션은 **두괄식** (핵심 → 설명 → 근거)으로 작성한다.

```
[핵심 메시지 1~2문장]

[상세 설명 2~3문단]
- 구체적 데이터
- 근거 자료
- 비교 분석

[표/차트/그림으로 요약]
```

### 배점 비례 분량
- 30점 이상: 3~5페이지
- 20~29점: 2~3페이지
- 10~19점: 1~2페이지
- 10점 미만: 0.5~1페이지

### 시각 자료 배치
- 서술 후 즉시 관련 표/차트 배치
- 표 제목은 상단, 그림 제목은 하단
- "[표 N]", "[그림 N]" 번호 부여
