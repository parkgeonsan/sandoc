| 2-1. 실현가능성 | 목표시장(고객) 분석 서술 |

## 6. HWP vs DOCX 대조

| 비교 항목 | HWP | DOCX |
|---------|-----|------|
| 파일 크기 | 69.0 KB | 44.7 KB |
| 포맷 | OLE2 Compound | ZIP/XML |
| 텍스트 추출 | zlib 해제 후 레코드 파싱 필요 | python-docx로 직접 접근 |
| 표 구조 | 레코드 기반 (파싱 복잡) | XML 기반 (28개 테이블 즉시 접근) |
| 스타일 정보 | DocInfo 스트림에 별도 저장 | 각 paragraph에 style 속성 포함 |
| 실무 활용성 | 파싱 난이도 높음, 별도 라이브러리 필요 | python-docx로 쉽게 활용 |

### 결론
- **HWP 직접 파싱**: 가능하나 복잡 (OLE2 → zlib 해제 → 레코드 파싱)
- **DOCX 우선 활용 권장**: 동일 양식의 .docx 버전이 제공되므로 DOCX로 처리하는 것이 효율적
- **HWP 전용 처리가 필요한 경우**: hwp5, pyhwp 등의 라이브러리 활용 또는 HWPX(한컴 최신 포맷) 변환 후 ZIP/XML로 처리
